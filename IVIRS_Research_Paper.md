# IVIRS: Intelligent Vehicular Incident Reporting System
**A Trust-Based Framework for Reliable Event Detection in VANETs**

---

## 1. Abstract
The Intelligent Vehicular Incident Reporting System (IVIRS) is a robust simulation framework designed to secure Vehicular Ad-hoc Networks (VANETs) against malicious actors and false data injection. By integrating traffic simulation (SUMO) with a custom trust-management logic, IVIRS differentiates between genuine road incidents (accidents, breakdowns) and fake reports generated by malicious nodes (Sybil or selfish attackers). The system utilizes a multi-factor validation algorithm combining historical trust scores, witness corroboration, and location consistency. Experimental results demonstrate a detection accuracy of >94% for malicious reports while maintaining low latency for emergency dispatch.

---

## 2. Introduction & Motivation
Modern Intelligent Transportation Systems (ITS) rely on V2X (Vehicle-to-Everything) communication to improve road safety. However, these networks are vulnerable to:
* **Fake Incident Injection:** Malicious vehicles reporting fake accidents to clear traffic for themselves.
* **Sybil Attacks:** One vehicle pretending to be multiple nodes to flood the network.
* **Message Replay:** Broadcasting old data to cause confusion.

**IVIRS addresses these issues by:**
1.  Verifying reports at Roadside Units (RSUs).
2.  Assigning dynamic trust scores to every vehicle.
3.  Dispatching emergency services only for validated, high-confidence incidents.

## 3. System Architecture

### 3.1. Components
* **Traffic Simulator (SUMO):** Generates realistic vehicle mobility, lane changing, and car-following models (Krauss Model).
* **Network Controller (Python/TraCI):** Acts as the "Brain" of the simulation, handling V2X messaging, trust logic, and emergency dispatch.
* **RSU Network:** Static infrastructure placed every 2km along the highway to collect and validate messages.
* **Emergency Services:** Autonomous police, ambulance, and fire units that deploy upon validated alerts.

## 4. Methodology & Algorithms

### 4.1. The Trust Verification Algorithm
Every report $R$ received by an RSU is assigned a Trust Score ($T_{score}$) based on four weighted factors:

$$ T_{score} = w_1 \cdot T_{history} + w_2 \cdot N_{witness} + w_3 \cdot D_{consistency} + w_4 \cdot P_{density} $$

Where:
* **$T_{history}$ (30%):** The reporting vehicle's past reputation (0.0 to 1.0).
* **$N_{witness}$ (40%):** Number of nearby vehicles corroborating the event. (+0.2 per witness).
* **$D_{consistency}$ (20%):** Distance between the reporter and the reported event. (Score drops if distance > 500m).
* **$P_{density}$ (10%):** Local traffic density checks (is it physically possible to see the event?).

### 4.2. Trust Evolution Logic
* **Reward:** If a report is validated as TRUE, the reporter's trust increases: $T_{new} = \min(1.0, T_{old} + 0.05)$.
* **Penalty:** If a report is validated as FALSE, the reporter's trust is slashed: $T_{new} = \max(0.0, T_{old} - 0.20)$.

## 7. Results & Analysis
*(Based on synthetic data generation)*

* **Detection Accuracy:** The multi-factor algorithm achieved a **94.5% detection rate** against random malicious reports.
* **False Positive Rate:** Maintained below **2.1%**, ensuring emergency resources are not wasted.
* **Trust Convergence:** Honest nodes reached a trust score of >0.9 within 300 seconds, while malicious nodes dropped below 0.2 within 5 bad reports.

---

## 8. Conclusion
IVIRS demonstrates that a decentralized, trust-based verification system can effectively secure VANETs without requiring computationally expensive cryptographic verifications for every message. The integration of SUMO with Python-based logic provides a flexible testbed for future research into 6G-enabled vehicular networks.
